RAG systems combine retrieval and generation. Retrieval finds relevant documents. Generation produces answers from retrieved context. Irrelevant documents can reduce answer quality.
Faithfulness measures whether generated answers are supported by retrieved documents. High faithfulness means answers are grounded in evidence. Low faithfulness means the answer may include hallucinated or unsupported claims.
Grounded generation requires the model to use only retrieved evidence. If evidence is missing, the model should abstain instead of guessing. Abstention is safer than returning an unsupported answer.
BM25 is a lexical retrieval algorithm. It relies on term frequency and inverse document frequency. BM25 is useful when the user query shares exact words with the document.
Dense retrieval uses embeddings to capture semantic similarity. It can retrieve documents even when the query and document use different words. Dense retrieval can miss exact keyword constraints.
Hybrid retrieval combines lexical retrieval and dense retrieval. It improves recall by searching with both exact word overlap and semantic similarity. Hybrid retrieval is often stronger than either method alone.
Cross-encoders rerank query-document pairs jointly. They are slower than bi-encoders but usually more accurate for final relevance ranking. Cross-encoders are useful after a broad retrieval stage.
Query rewriting removes ambiguity from user questions. Query expansion creates multiple search variants. Step-back queries capture the broader concept behind a question.
Context compression reduces token usage by keeping only relevant document sentences. Compression can improve generation quality by removing distracting context. Compression must preserve facts needed for the answer.
Answer planning breaks a question into sub-questions and expected claims. Planning helps verify whether enough evidence exists before generation. Planning can prevent the model from answering when evidence is incomplete.
Claim-aware generation restricts the model to supported claims. A claim should only appear in the answer if the evidence directly supports it. Unsupported generated claims should be filtered or rejected.
A faithfulness judge evaluates each answer claim against retrieved evidence. Claims may be supported, partially supported, unsupported, or contradicted. Contradicted claims should force low confidence.
An abstention threshold decides whether to accept, retry, or reject an answer. If the faithfulness score is high and no hard failures exist, the answer can be accepted. If the score remains low after retries, the system should answer I don't know.
Retrieval self-correction uses feedback from failed retrieval attempts. It can rewrite the query using missing information, unsupported sub-questions, and low relevance scores. This helps recover from poor initial search results.
Generation self-correction uses feedback from the faithfulness judge. The planner receives the previous answer, previous claims, and judge analysis. It replans to avoid repeating unsupported or contradicted claims.
VeriRAG uses a maximum of two retrieval retries. VeriRAG also uses a maximum of two faithfulness retries. Retry limits prevent infinite correction loops.
VeriRAG treats a faithfulness score of 0.85 as the passing threshold. Answers below this threshold are retried when retries remain. Answers below this threshold after retries are abstained.
VeriRAG combines claim faithfulness, evidence coverage, and retrieval confidence into a final confidence score. This gives a broader signal than retrieval similarity alone. Confidence labels can be high, medium, low, or abstained.
LangGraph represents the RAG workflow as nodes and conditional edges. Each node updates graph state. Conditional edges decide whether to retrieve again, generate, retry, accept, or abstain.
LangSmith tracing can help debug graph execution. Traces show prompts, intermediate state, model responses, and routing decisions. Tracing is useful during interview demonstrations and production debugging.
